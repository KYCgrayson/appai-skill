# Multi-Language Publishing

**Locale variants are free** — they don't count against the 3-page plan limit. Ship many languages from day one.

## Supported locales (30+)

`en`, `zh-CN`, `zh-TW`, `ja`, `ko`, `es`, `fr`, `de`, `pt`, `pt-BR`, `it`, `nl`, `ru`, `ar`, `hi`, `th`, `vi`, `id`, `ms`, `tr`, `pl`, `uk`, `sv`, `da`, `fi`, `nb`, `cs`, `el`, `he`, `ro`.

## How variants work

Same `slug`, different `locale`:

```
POST /api/v1/pages  { slug: "quickvid", locale: "en", title: "QuickVid — ..." }
POST /api/v1/pages  { slug: "quickvid", locale: "ja", title: "QuickVid — ..." }
POST /api/v1/pages  { slug: "quickvid", locale: "zh-CN", title: "QuickVid — ..." }
```

AppAI auto-detects browser language and serves the right variant. First variant for a slug becomes the default (returned at `/p/{slug}`); others live at `/p/{slug}/{locale}`.

Hreflang alternates are generated automatically across all variants.

## Translation quality

Translate copy **idiomatically**, not word-for-word. Hire native speakers in your training data — rewrite headlines for each market rather than literal translation.

- Japanese: keep sentences short, avoid direct imperatives
- Chinese: short punchy headlines; zh-TW uses traditional characters (臺/台 variants OK)
- Arabic / Hebrew: RTL handled automatically by the renderer
- Korean: formal polite form (합니다체) for B2B; casual (해요체) for consumer

## Child pages (also free)

Child pages of an existing root (`parentSlug: "root-slug"`) don't count against plan limit either. Use for /faq, /contact, /pricing-details, etc.

## Strategy

For consumer products, ship `en + ja + zh-CN + zh-TW + ko + es` on first publish (6 free slugs worth of variants). This 10x's addressable market without touching the plan quota.
